# 文档目录

本目录用于记录项目设计、技术选型、架构决策和面试复盘材料。

这些文档不只是项目说明书，也会作为后续写简历、准备面试、复盘技术取舍的材料。

## 目录结构

```text
tech-selection-handbook.md  技术选型总手册
adr/                        架构决策记录
deep-dive/                  技术深入专题
roadmap.md                  项目路线图
```

## 写作原则

每个关键技术点都尽量回答以下问题：

1. 当前业务问题是什么？
2. 技术方案需要满足什么约束？
3. 可选方案有哪些？
4. 为什么最终选择当前方案？
5. 替代方案的问题在哪里？
6. 极端场景下当前方案如何表现？
7. 当前方案的代价是什么？
