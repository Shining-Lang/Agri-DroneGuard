# 技术选型总手册

本手册独立于具体代码实现，用于记录项目中每个关键技术栈的选择原因、替代方案、极端场景表现和面试表达。

它的目标不是堆技术名，而是沉淀工程判断：为什么选这个技术，不选其他方案会有什么问题，它在高并发、失败恢复、安全审批、上线部署等场景下如何兜底。

## 选型总原则

1. Java 作为可信业务底座。
2. Python Agent 负责 AI 规划和工具编排。
3. GIS 校验和确定性算法负责安全边界。
4. 大模型只做辅助决策，不直接执行高风险操作。
5. 高风险任务必须有人审闭环。
6. 本地开发和在线演示都应该可以通过 Docker Compose 复现。

## 标准分析模板

每个重要技术栈都按以下问题展开：

1. 当前业务问题是什么？
2. 技术方案需要满足什么约束？
3. 可选方案有哪些？
4. 为什么最终选择当前方案？
5. 替代方案的问题在哪里？
6. 极端场景下当前方案如何表现？
7. 当前方案的代价是什么？
8. 在本项目中的落点是什么？
9. 面试时怎么讲？
