# 项目路线图

本路线图用于控制项目范围，避免一开始把系统做得过大。每个阶段都应该能交付一个可以演示、可以解释、可以继续扩展的结果。

## 阶段 0：项目地基

- 创建 Monorepo 目录结构。
- 创建 `backend`、`agent`、`frontend`、`infra`、`docs` 基础骨架。
- 明确项目边界：只做演示级/准生产级系统，不控制真实无人机。
- 建立技术选型文档、ADR 和 deep-dive 文档体系。

## 阶段 1：本地运行环境

- 使用 Docker Compose 启动 PostgreSQL/PostGIS。
- 使用 Docker Compose 启动 Redis。
- 跑通 Spring Boot 后端健康检查接口。
- 跑通 FastAPI Agent 健康检查接口。
- 跑通 Vue3 前端页面。

## 阶段 2：核心业务闭环

- 建立打药任务订单模型。
- 建立地块、禁飞区、审批、航线、审计日志等基础表。
- Java 后端创建任务并投递到 Redis Stream。
- Python Agent 消费规划任务。
- Java 后端接收并持久化 Agent 规划结果。

## 阶段 3：Agent 规划能力

- 构建 LangGraph 工作流。
- 增加农艺知识 RAG 占位能力。
- 增加确定性航线生成工具。
- 增加空间风险校验工具。
- 增加风险评分与结构化输出校验。

## 阶段 4：人工审批与模拟执行

- 高风险任务自动挂起。
- 管理员可以同意或驳回审批。
- 使用 SSE 输出模拟执行日志。
- 生成模拟无人机飞控任务包。
- 生成任务执行报告或 JSON 结果。

## 阶段 5：上线演示

- 整理生产演示用 Docker Compose。
- 增加 Nginx 反向代理。
- 增加初始化演示数据。
- 增加演示账号。
- 完善 README、技术选型文档和简历表达。
